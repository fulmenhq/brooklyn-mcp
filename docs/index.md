## Contributing to Documentation

Brooklyn documentation follows structured patterns:

1. **User-focused** - Written for specific audiences and use cases
2. **Implementation-verified** - All code examples tested and working
3. **Cross-linked** - Documents reference related content
4. **Maintained** - Regular review and updates

To contribute:

1. Identify your audience (user, contributor, implementer, administrator)
2. Follow existing document patterns in the relevant section
3. Include working code examples and diagrams where helpful
4. Submit PR with clear description of documentation changes
